> **Этот репозиторий — не просто код, а демонстрация моего подхода к написанию промышленных автотестов на Python.**  
> Здесь я показываю, как строить чистые, читаемые и информативные тесты для объектно‑ориентированной системы, используя лучшие практики и современные инструменты.

## 👀 О чём этот проект?

Я спроектировал небольшую, но показательную **систему управления библиотекой**: классы `Book`, `User` и `Library` с бизнес‑логикой (добавление/удаление книг, регистрация пользователей, выдача и возврат, поиск по названию/автору/ISBN).  
**Цель проекта — не сама библиотека, а архитектура тестов**, которая демонстрирует:

- Как использовать **фикстуры** (`fixtures`) для подготовки тестового окружения вместо громоздкого `setUp`.
- Как выносить все фикстуры в `conftest.py` для переиспользования и чистоты тестов.
- Как тестировать **исключения** (`ValueError`, `RuntimeError`) и граничные случаи.
- Как **документировать тесты** через декораторы Allure (`@allure.feature`, `@allure.story`, `@allure.title`, `with allure.step`).
- Как превратить консольный вывод `pytest` в красивый **Allure‑отчёт** с группировкой по функциональности, шагами выполнения и графиками.

## 🛠 Технологический стек и чему я научился

| Инструмент | Зачем я его использовал? |
|------------|--------------------------|
| **Pytest** | Основной фреймворк. Пишу компактные и выразительные тесты. |
| **pytest‑cov** | Измеряю покрытие кода, чтобы видеть «слепые зоны». В этом проекте бизнес‑логика покрыта на 100%. |
| **Allure Framework** | Генерирую отчёты, которые понятны даже менеджеру или заказчику: тесты сгруппированы по фичам, есть пошаговые логи. |
| **Фикстуры (Fixtures)** | Управляю тестовым окружением: создаю книги, пользователей, библиотеки с предустановленными данными. |
| **conftest.py** | Все фикстуры вынесены в центральный файл — тесты стали чище и легче поддерживаются. |

## 🚀 Как это запустить и посмотреть магию?

Если хотите увидеть всё своими глазами, следуйте инструкции ниже.

### 1. Подготовка (один раз)

```bash
# Клонируем репозиторий (или создаём файлы вручную)
# Создаём виртуальное окружение и ставим всё необходимое
python -m venv venv

# Просто запустить тесты с подробным выводом
pytest -v

# Собрать данные для Allure-отчёта
pytest --alluredir=./allure-results

# Сгенерировать и открыть Allure-отчёт (откроется браузер)
allure serve ./allure-results

# Увидим процент покрытия и какие строки я пропустил (спойлер: 100% 😉)
pytest --cov=library --cov-report=term-missing

📸 Пример того, что вы увидите в Allure
Вместо скучной «простыни» в консоли вы получите дашборд:

Behaviors → Тесты сгруппированы по фичам: Управление книгами, Управление пользователями, Система библиотеки, Поиск, Пустая библиотека.

Suites → Техническая группировка по файлу test_library.py.

Graphs → Графики длительности выполнения и стабильности.

Пошаговые логи → На каждый тест можно нажать и увидеть, что происходило внутри (например, «Проверка атрибутов книги», «Попытка добавить дубликат»).
